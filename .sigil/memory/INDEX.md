# Knowledge Index

## project.md
High-level overview of what Sigil is, who it's for, and how it works. Covers the full project structure (every file in `sigil/` and `tests/`), the Python/uv/litellm/PyGithub stack, all CLI commands, build/test/lint instructions, business model phases, current status (Phase 1 MVP complete, dogfooding done, knowledge compaction rewritten, agent config detection implemented), hard rules (never commit without `/commit-review`, always run `ruff format` last), known bugs, and the issue tracker location. Read this when starting any task — it is the essential orientation file. Also read when you need to know how to run tests or lint, where files live, or what the project's constraints and rules are.

## architecture.md
Complete architectural reference for the Sigil pipeline. Covers the full async pipeline from `sigil run` through discovery, agent config detection, knowledge compaction (now single-call JSON with INIT/INCREMENTAL modes), analysis+ideation (parallel), validation, dedup, parallel execution, PR/issue publishing, cleanup, and working memory update. Each module gets a detailed responsibility breakdown including the new `agent_config.py` module and how agent configs are injected into all agent prompts. Also covers the async model, full data flow diagram, and key design principles. Read this when working on any module interaction, pipeline ordering, or when you need to understand how components connect.

## api.md
Exhaustive API reference for every public function, dataclass, and constant in Sigil. Covers all data structures including the updated `Config` (with new `fast_model` and `knowledge_model` fields). Documents the rewritten `compact_knowledge` API (INIT vs INCREMENTAL modes, JSON response format, skip-when-HEAD-matches behavior). Lists all LLM tool schemas including the new `read_knowledge_file` tool and the removal of `write_knowledge_file`. Documents the new `detect_agent_configs()` function in `agent_config.py`. Lists all constants including new `knowledge.py` constants and `agent_config.py` constants. Read this when implementing or modifying any function, when you need exact field names and types, or when debugging tool-call schemas.

Read this before writing any new code — it defines what correct Sigil code looks like.

Read this when working on `knowledge.py`, `memory.py`, the compaction pipeline, or when diagnosing stale knowledge issues.
