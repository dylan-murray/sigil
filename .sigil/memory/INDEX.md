# Knowledge Index

## project.md
High-level overview of what Sigil is, who it's for, and how it works. Covers the project's purpose (autonomous repo improvement agent), target users, language/stack (Python 3.11+, uv, typer, litellm, PyGithub, asyncio), project directory structure, business model phases, current status (108 tests, Phase 1 MVP complete), and hard project rules (never commit without review, always run ruff format last, no comments unless necessary). Read this when starting any task, orienting to the codebase, or needing to understand project constraints and conventions.

## architecture.md
Detailed breakdown of the full async pipeline and every component's responsibilities. Covers the end-to-end flow from `sigil run` through discovery, knowledge compaction, analysis, ideation, validation, deduplication, parallel execution, PR/issue publishing, cleanup, and working memory update. Each module (`cli.py`, `config.py`, `discovery.py`, `knowledge.py`, `memory.py`, `maintenance.py`, `ideation.py`, `validation.py`, `executor.py`, `github.py`, `llm.py`, `utils.py`) gets a dedicated section explaining what it does and how it fits in. Also covers the async model (litellm, asyncio, to_thread for PyGithub) and key design principles. Read this when working on any module, tracing data flow, or understanding how components interact.

## api.md
Complete API reference for all public functions, data structures, constants, and LLM tool schemas. Covers every dataclass (`Finding`, `FeatureIdea`, `ExecutionResult`, `Config`, `GitHubClient`, `DedupResult`, `ValidationResult`, `WorkItem`, `_ChangeTracker`) with field-level documentation, all public functions organized by module with signatures and behavior notes, all LLM tool schemas (knowledge tools, analysis tools, validation tool, executor tools), and all constants (timeouts, limits, thresholds). Also includes a Known Issues section listing current bugs and gaps. Read this when implementing or modifying any function, checking field names/types, understanding tool schemas, or looking up constants.

## patterns.md
Coding conventions and implementation patterns used throughout the codebase. Covers Python standards (3.11+, PEP 604 union types, no bare excepts, no comments, 100-char lines, double quotes), naming conventions (snake_case, PascalCase, UPPER_SNAKE_CASE, leading underscore for private), frozen dataclass pattern with `dataclasses.replace()`, the standard LLM tool-use loop pattern (messages list, MAX_LLM_ROUNDS, tool_call handling, finish_reason check), async subprocess via `arun()`, parallel execution with `asyncio.gather` and `Semaphore`, GitHub API wrapping with `asyncio.to_thread`, path safety validation, config loading pattern, memory/knowledge file access via `read_file()`, error handling philosophy, import organization, slug/branch naming, prompt structure, and validation item indexing. Read this before writing any new code, when unsure about conventions, or when implementing LLM interactions.

## configuration.md
User-facing configuration reference for `.sigil/config.yml`. Covers the full YAML schema with all fields and defaults, strict validation rules (unknown fields raise ValueError, boldness enum validation, version stripping), detailed explanation of all four boldness levels (conservative/balanced/bold/experimental) and their effect on analysis scope and feature ideation, all six focus areas (tests, dead_code, security, docs, types, features), required environment variables (API keys, GITHUB_TOKEN), supported model formats for Anthropic/OpenAI/Google via litellm, all CLI commands including removed ones, auto-detection behavior for lint/test commands, memory and ideas directory structure, and Sigil's own config as a concrete example. Read this when modifying config loading, adding new config fields, working on CLI commands, or understanding how boldness/focus affect agent behavior.

## execution-model.md
Deep dive into how Sigil executes code changes using git worktrees. Covers the full worktree lifecycle (create → execute → commit → rebase → push → cleanup), the code generation tool-use loop with `apply_edit`/`create_file`/`read_file`/`done` tools and their constraints, the lint→test→retry loop with error feedback to LLM, rollback on failure via git checkout and file deletion, the five failure downgrade triggers, parallel execution with Semaphore and slug deduplication, memory conflict auto-resolution during rebase (take main's version), the `ExecutionResult` interpretation table, cleanup strategy for pushed vs failed branches, and command timeouts. Also documents the known `execute_parallel` sentinel bug (`""` vs `str | None`). Read this when working on `executor.py`, debugging worktree issues, understanding failure/downgrade behavior, or modifying the retry/rollback logic.

## github-integration.md
Complete reference for GitHub API integration. Covers authentication requirements (GITHUB_TOKEN, required permissions, fail-fast behavior), repository detection from git remote (SSH and HTTPS URL parsing), the three-strategy deduplication system (exact title match, category+file key, Jaccard token similarity ≥ 0.6), PR creation flow with body template, issue creation flow with body template (findings, ideas, downgrade context), label management (sigil label + sigil:{category} labels, auto-creation), rate limiting via tenacity retry decorator on sync functions, graceful degradation for all failure modes, publishing limits enforcement (max_prs_per_run, max_issues_per_run), branch cleanup after push, GitHub Actions integration example with required permissions and fetch-depth, and the async wrapping pattern for PyGithub sync calls. Read this when working on `github.py`, debugging PR/issue creation, modifying deduplication logic, or setting up GitHub Actions.

## knowledge-system.md
Explains Sigil's persistent knowledge architecture — how raw repo discovery is compacted into structured markdown files that agents selectively load. Covers the `.sigil/memory/` directory structure, staleness detection via HEAD SHA in INDEX.md HTML comment, the compaction flow (`compact_knowledge` — LLM writes files via tool calls, budget system, reserved file protection), knowledge selection flow (`select_knowledge` — LLM reads INDEX.md and picks relevant files), INDEX.md format with HEAD SHA comment, working memory (`working.md`) managed separately by `memory.py` with LLM compaction under 100 lines, ideas storage in `.sigil/ideas/` with YAML frontmatter and TTL cleanup, memory snapshot copying into worktrees at creation time, security warning about public knowledge files, and file naming conventions. Read this when working on `knowledge.py` or `memory.py`, debugging stale knowledge, understanding how agents get context, or modifying the compaction/selection pipeline.
