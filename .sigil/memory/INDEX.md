# Knowledge Index

## project.md
High-level overview of what Sigil is, who it's for, and how it works. Covers the full project structure (every file in `sigil/`, `tests/`, `.github/workflows/`, `examples/`, `.issues/`), the Python/uv/litellm/PyGithub/mcp stack, all CLI commands, build/test/lint instructions, current status (Phase 1 + extensibility + quality tracks all complete, Sprint 7 done — parallel validation + per-agent model config, dogfood CI added), hard rules (never commit without `/commit-review`, always run `ruff format` last, copy industry patterns), known bugs, and the issue tracker location. The `.github/workflows/` section now includes `sigil.yml` (daily dogfood run). Read this when starting any task — it is the essential orientation file. Also read when you need to know how to run tests or lint, where files live, or what the project's constraints and rules are.

## architecture.md
Complete architectural reference for the Sigil pipeline. Covers the full async pipeline from `sigil run` through discovery, agent config detection, knowledge compaction, MCP server connection, analysis+ideation (parallel), validation (single mode or parallel mode with two reviewers + arbiter), dedup, parallel execution, PR/issue publishing, cleanup, and working memory update. Each module gets a detailed responsibility breakdown including `mcp.py` (MCP client with deferred tool loading, `mcp__server__tool` naming, `prepare_mcp_for_agent`, `handle_search_tools_call`), updated `validation.py` (single vs parallel mode, per-agent model routing for reviewer/arbiter), and updated `config.py` (`validation_mode`, `fast_model` removed, `agents` dict). Also covers the async model, full data flow diagram, module table, and key design principles. Read this when working on any module interaction, pipeline ordering, or when you need to understand how components connect.
