# Knowledge Index

## project.md
High-level overview of what Sigil is, who it's for, and how it works. Covers the full project structure (every file in `sigil/` and `tests/`), the Python/uv/litellm/PyGithub stack, all CLI commands, build/test/lint instructions, business model phases, current status (108+ tests, Phase 1 MVP complete), hard rules (never commit without `/commit-review`, always run `ruff format` last), known bugs, and the issue tracker location. Read this when starting any task — it is the essential orientation file. Also read when you need to know how to run tests or lint, where files live, or what the project's constraints and rules are.

## architecture.md
Complete architectural reference for the Sigil pipeline. Covers the full async pipeline from `sigil run` through discovery, knowledge compaction, analysis+ideation (parallel), validation, dedup, parallel execution, PR/issue publishing, cleanup, and working memory update. Each module (`cli.py`, `config.py`, `discovery.py`, `knowledge.py`, `memory.py`, `maintenance.py`, `ideation.py`, `validation.py`, `executor.py`, `github.py`, `llm.py`, `utils.py`) gets a detailed responsibility breakdown. Also covers the async model (litellm async, asyncio subprocess, PyGithub via `to_thread`, Semaphore concurrency), the full data flow diagram, and key design principles (conservative by default, CI must pass, small PRs, tool-use pattern). Read this when working on any module interaction, pipeline ordering, or when you need to understand how components connect.

## api.md
Exhaustive API reference for every public function, dataclass, and constant in Sigil. Covers all data structures (`Finding`, `FeatureIdea`, `ExecutionResult`, `Config`, `GitHubClient`, `DedupResult`, `ValidationResult`, `WorkItem`, `_ChangeTracker`) with full field definitions and types. Lists every public function per module with signatures and behavior notes. Documents all LLM tool schemas (knowledge tools, analysis tools, validation tool, executor tools). Lists all constants (`MAX_TOOL_CALLS_PER_PASS`, `COMMAND_TIMEOUT`, `MAX_LLM_ROUNDS`, `SIMILARITY_THRESHOLD`, etc.) and known issues/gaps. Read this when implementing or modifying any function, when you need exact field names and types, or when debugging tool-call schemas.

## patterns.md
Coding conventions and implementation patterns used throughout Sigil. Covers Python standards (3.11+, PEP 604 unions, no bare excepts, no comments unless necessary, ruff line length/quotes), naming conventions (snake_case, PascalCase, UPPER_SNAKE_CASE, leading underscore for private), frozen dataclass pattern with `dataclasses.replace()`, the standard LLM tool-use loop (messages list, appending assistant message before tool responses, `MAX_LLM_ROUNDS=10`, always import `acompletion` from `sigil.llm`), async subprocess via `arun()`, parallel execution with `asyncio.gather` and `Semaphore`, GitHub API wrapping with `asyncio.to_thread`, path safety validation, config loading pattern, memory/knowledge file access via `read_file()`, error handling philosophy, import organization, slug/branch naming, prompt structure, and validation item indexing. Read this before writing any new code — it defines what correct Sigil code looks like.

## execution-model.md
Deep dive into how Sigil executes code changes using git worktrees. Covers the full worktree lifecycle (create → execute → commit → rebase → push → cleanup), the LLM code generation tool loop (`read_file`, `apply_edit`, `create_file`, `done` with `MAX_TOOL_CALLS_PER_PASS=15`), `apply_edit` constraints (exact match, uniqueness, path validation, known empty `old_content` gap), the `_ChangeTracker` dataclass, the lint→test→retry loop with error feedback, rollback on failure, the five downgrade triggers, parallel execution with `asyncio.Semaphore`, slug deduplication, memory conflict auto-resolution during rebase (`--ours` for `.sigil/memory/` files), code conflict downgrade, `ExecutionResult` interpretation table, cleanup strategy, and command timeouts. Read this when working on `executor.py`, worktree management, branch strategy, or diagnosing execution/rebase failures.

## github-integration.md
Complete reference for Sigil's GitHub integration. Read this when working on `github.py`, PR/issue creation, deduplication logic, or GitHub Actions setup.
